# 为每个源代码文件创建独立文档的计划

## 目标
为 AutoSpider 项目的每个 Python 源代码文件创建独立的文档，文档风格参考 `prompt 模板渲染工具--prompt_template.py.md`。

## 文档结构
按照项目模块结构组织文档：

### Common 模块 (6 个文档)
- `config.py.md` - 配置管理
- `types.py.md` - 核心数据类型定义
- `logger.py.md` - 日志记录
- `validators.py.md` - 验证器
- `constants.py.md` - 常量定义
- `exceptions.py.md` - 异常定义

### Browser 子模块 (2 个文档)
- `session.py.md` - 浏览器会话管理
- `actions.py.md` - 动作执行器

### SoM 子模块 (1 个文档)
- `api.py.md` - SoM Python API

### Storage 子模块 (2 个文档)
- `persistence.py.md` - 持久化基类
- `redis_manager.py` - Redis 管理器

### Crawler 模块 (4 个文档)
- `url_collector.py.md` - URL 收集器
- `batch_collector.py.md` - 批量爬取控制器
- `base_collector.py.md` - 基础收集器
- `checkpoint/rate_controller.py.md` - 自适应速率控制器
- `checkpoint/resume_strategy.py.md` - 断点恢复策略

### Extractor 模块 (12 个文档)
- `config_generator.py.md` - 配置生成器
- `llm/prompt_template.py.md` - Prompt 模板渲染
- `llm/planner.py.md` - LLM 规划器
- `llm/decider.py.md` - LLM 决策器
- `graph/agent.py.md` - Agent 图
- `collector/xpath_extractor.py.md` - XPath 提取器
- `collector/url_extractor.py.md` - URL 提取器
- `collector/pagination_handler.py.md` - 分页处理器
- `collector/navigation_handler.py.md` - 导航处理器
- `collector/llm_decision.py.md` - LLM 决策器
- `collector/page_utils.py.md` - 页面工具
- `collector/models.py.md` - 数据模型
- `output/script_generator.py.md` - 脚本生成器
- `validator/mark_id_validator.py.md` - mark_id 验证器

### 其他 (2 个文档)
- `cli.py.md` - 命令行接口
- `__main__.py.md` - 主入口

## 文档格式
每个文档包含：
1. **标题** - 文件名和简短描述
2. **文件路径** - 源代码文件路径
3. **核心功能** - 主要类和函数
4. **函数目录** - 按功能分类的函数/方法列表
5. **使用示例** - 完整的代码示例
6. **最佳实践** - 使用建议和注意事项
7. **故障排除** - 常见问题和调试技巧

## 文档存储位置
- 文档保存在 `docs/` 目录下
- 按模块组织子目录，如 `docs/common/`, `docs/crawler/` 等
- 文件名与源代码文件名对应，如 `config.py.md`

## 总计
约 30 个文档文件需要创建或更新。
