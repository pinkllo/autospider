# Crawler Explore 模块

该模块实现了详情页 URL 的自动探索与模式识别逻辑。

## 核心组件

### `URLCollector` 
这是整个系统的“协调器”，负责执行从零开始的自动化探索流程。

#### 三阶段工作流：
1. **探索阶段 (Explore)**：
   - 在 LLM 的引导下，尝试点击页面上的不同链接进入详情页。
   - 记录进入详情页所需的每一个操作步骤（导航路径）。
2. **分析阶段 (Analyze)**：
   - 分析多次进入详情页的路径和 URL。
   - 使用 `XPathExtractor` 自动提取详情页链接的公共 XPath 模式。
3. **收集阶段 (Collect)**：
   - 使用提取出的 XPath 模式，结合 `PaginationHandler` 高效遍历所有列表页。
   - 收集并保存所有符合条件的详情页 URL。

### `ConfigGenerator` 
负责将探索阶段得到的模式和步骤生成可复用的配置文件，供 `BatchCollector` 使用。

## 技术特点
- **LLM 引导**：利用语义理解识别详情页，无需预定义规则。
- **自动模式识别**：将慢速的 LLM 决策转化为高速的 XPath 规则。
- **端到端流程**：从初始 URL 到完整的采集配置自动化生成。
